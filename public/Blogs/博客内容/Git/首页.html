Git 和 GitHub

  • Git 和 GitHub 是两回事

    • Git 是版本控制工具，是一个系统软件。
    • GitHub 是代码托管平台，是一个服务网站。
    • 即使没有 GitHub，Git 也完全可以正常运作。

  • Git 是核心，GitHub 是在 Git 的基础上创建起来的服务网站。

    • Git 最初由 Linus Torvalds 开发，并在2005年发布了第1版。
    • Git 最初的目的是为了解决 Linux 内核开发过程中的问题。
    • GitHub 是由 Tom Preston-Werner、 Chris Wanstrath、 P. J. Hyett 以及 Scott Chacon 在2008年4月正式推出的服务平台，专门为世界各地的程序开发者提供代码托管服务，现今已经成为全球最大的程序员社区。绝大部分最重要、最流行的开源软件，源代码都托管在 GitHub 上。

  • 为什么要使用 Git ？

    • 追溯每一次代码更新的详情：

      • 更新了哪些内容？增加、修改、删除
      • 谁修改的？用户名、邮箱
      • 什么时间修改的？

    • 在不同分支上开发项目：

      • 想试验一个新功能，但又怕把项目搞得乱七八糟，怎么办？新开一个分支吧！不影响原有分支！
      • 新开的分支写得太垃圾了，不想要了，咋办？直接删了就行了！
      • 我以为新代码写得不错，但过了一阵子感觉还是不行，能反悔吗？可以的！能够倒回到某个节点！

    • 精确地管理多人一起开发项目的进度状态。

      • 每个人在各自的分支上写代码，互不影响。
      • 谁修改了哪一句代码，全部可追朔。

    • 分布式，每个人的本地仓库中都可以拥有完整代码。

    • 现代开发者必备技能。

  • 为什么要使用 GitHub ？

    • 想寻找开源免费的软件？来 GitHub 吧！

    • 希望在最大的程序员社交平台展示自己的作品？来 GitHub 吧！

    • 希望别人也能参与到你的项目开发中来？来 GitHub 吧！

    • 希望参与别人的项目？来 GitHub 吧！
