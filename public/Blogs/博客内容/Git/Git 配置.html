使用 Git Config 命令进行配置

  • 配置用户名

    • git config --global user.name "用户名"
    • --global 代表全局设置，只需设置一次，今后所有的Git项目都会自动使用这个用户名。如果不使用这个参数，则用户名仅对当前仓库有效，每个Git仓库都要单独设置用户名。
    • 尽管用户名可以随便写，但最好还是和你代码托管平台例如 GitHub的用户名保持一致，以免推送时出现问题。

  • 配置用户邮箱 

    • git config --global user.email "用户邮箱"
    • --global 代表全局设置，只需设置一次，今后所有的Git项目都会自动使用这个邮箱。如果不使用这个参数，则邮箱仅对当前仓库有效，每个Git仓库都要单独设置邮箱。
    • 尽管邮箱可以随便写，但最好还是和你代码托管平台例如 GitHub的邮箱保持一致，以免推送时出现问题。
