Git 下载-安装-更新原创

作者： 苏扬

日期： 2023年7月6日

下载 Git

访问Git官方网站

点击 Git 官方网站链接：

点击Download for Windows即可下载最新版本。

提醒

Git 原本是 Linux 上的系统软件，后来才出现了独立的 Windows 版本，称之为 Git for Windows，有独立的官方网站。如果官网太慢，也可以访问镜像站点，选择对应版本下载。

安装 Git

用户协议

这是用户协议，主要陈述了跟版权、使用权有关的事项，点击Next就代表同意。

安装路径

根据自己的需要进行更改，无特殊需求就用默认路径。

选择组件

无特殊需求就用默认设置。

开始菜单

可自定义开始菜单文件夹的名称，也可点选Don't create a Start Menu folder，这样就不会创建开始菜单文件夹。

默认编辑器

默认的是Vim。

提醒

你会发现这个界面上有两个Note提醒，第1个提醒说：Vim 只是因为历史遗留原因，才成为了 Git 的默认编辑器，强烈建议另选一个拥有现代化界面的替代品我选择了Visual Studio Code。

调整新仓库的初始分支名称

如果选择Let Git decide，则新仓库初始分支的名称默认为master；如果想自定义名称，请选择Override the default branch name for new repositories，目前用得比较多的名称是main。

小知识

2020年6月12日，GitHub决定将新仓库的初始分支默认名称从master改为main。此事的缘由，是因为越来越多的国外开发者觉得master直译为 “主人”具有冒犯性，词汇带有奴隶制的特征，因此要求修改，这也算是国外的一种政治正确。
其实，master主，直译：主人和slave从，直译：仆人配套的命名方式，早在笔者20世纪90年代初学电脑时就已存在很多年了，当时电脑上如果有多个硬盘和光盘，需要设置哪个是主磁盘，以便系统能够正确引导。此概念如今已经成为历史。

调整环境变量

根据你想要调用Git的命令行工具，选择添加到环境变量中的路径。没有特殊需求的话，默认即可。

选择 SSH 客户端

默认选项是使用 Git 自带的 SSH。另一个是使用外部 SSH，默认即可。

选择 HTTPS 后端传输协议库

默认即可。

配置行末尾转换

默认即可。

配置 Git Bash 使用的终端模拟器

默认的MinTTY功能比较丰富，因此默认即可。

选择 Git Pull 的默认行为

默认即可。

选择凭据助手

默认即可。

配置附加选项

默认即可。

配置实验性选项

实验性选项是给喜欢折腾的开发者用的，就不用选了。

完成安装

最后一步了，想要完成后立刻开始使用Git Bash的，可以勾选第1个选项。

更新 Git

直接下载 Git，并手动安装

  1. Git 官网
  2. Git for Windows 官网
  3. npm 镜像站

用命令更新 Git

在终端运行命令：git update-git-for-windows本质也是在后台自动下载 Git，速度可能较慢，还可能连不上。
