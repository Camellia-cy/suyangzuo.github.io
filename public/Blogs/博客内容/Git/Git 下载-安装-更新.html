初次接触 Git❔ 你来对地方了❕

  • 下载 Git

    • 访问 Git 官网，点击专门的 Windows 版本链接。
    • Git 原本是 Linux 上的系统软件，后来才出现了独立的 Windows 版本，称之为 Git for Windows，有独立的官网。
    • 如果官网太慢，也可以访问镜像站点，选择对应版本下载。

  • 安装 Git

    1 用户协议，没什么好说的，不同意就无法安装。

    2 安装路径，没什么特殊需求就不用改。

    3 选择模块，没有特殊需求的话，默认就可以。

    📢 Git Bash 是一个专门为 Git 开发的命令行工具，可以直观地提供 Git 使用中的信息。但如果你使用了 Oh My Posh 的话，很大程度上可以替代 Git Bash。建议装起来，以备不时之需；并且将其添加到 Windows 终端 中，以统一界面。

    4 是否创建开始菜单。

    5 选择 Git 使用的默认编辑器。

    📢 默认的编辑器是 Vim。注意，第1个提醒说到：Vim 只是因为历史遗留原因，才成为了 Git 的默认编辑器，强烈建议另选一个拥有现代化界面的替代品。

    6 选择初始分支的名称。

    📢 这里有一个很有意思的事情：长期以来，初始分支的名称都是master。然而，由于欧美国家过去长期的殖民侵略历史，有人觉得用master不合适，因为master主人对应的词汇是slave奴隶，所以现在很多项目都将主分支的名称改成了main主要的。

    欧美国家特别喜欢干这种浮于表面，彰显自己道德标杆，又不用付出真金白银的事情，比如举标语、黑名贵、各种环保抗议等。然而当真的要他们放弃传统污染能源、踏踏实实种树、归还掠夺宝物时，一个个开始找各种冠冕堂皇的理由，其虚伪无耻可谓叹为观止。所以，main 还是 master，随便！

    7 调整环境变量，选中间的推荐选项即可。

    8 选择 SSH 客户端，选第1个集成的即可。

    9 HTTPS 传输协议库选择，第1个即可。

    10 选择怎样处理回车换行符，我们用第1种就行。

    11 选择终端模拟器，选第1个就行，反正后面都用Windows终端。

    12 选择 git pull 的默认行为，默认选项即可。

    13 选择凭据管理器，选第1个即可。

    14 附加选项，保持默认不变即可。

    15 实验性选项，既然是实验性，说明还不完善，就别选了。

    16 安装完成，第1个选项是完成后打开 Git Bash，第2个选项是完成后查看发布说明，自己看着办，我都去掉了。

  • 更新 Git

    📢 要更新Git，有2种方法：

    • 下载新版本的 Git，再次安装。

    • 使用命令行更新：git update-git-for-windows 本质也是在后台自动下载 Git，速度可能较慢，还可能连不上。
