掌握 Git 的基础使用流程

  • Git 工作流

    如图所示，Git 工作流整体上分为2部分：

    • 本地仓库 图中白色部分，离线也可正常工作。
    • 远程仓库 图中灰色部分，必须联网，所谓远程仓库就是托管在 GitHub 或者 Gitee 之类的平台上的仓库。

    让我们暂时聚焦于本地，只要关注下图所涵盖的范围即可：

  • 初始化 Git 本地仓库

    • git init
    • 在你的项目文件夹工作区输入如上命令，会将当前文件夹转化为一个Git 本地仓库，并在其中创建一个隐藏的 .git 子文件夹，里面保存了项目历史记录所需要的所有数据；与此同时，Git 会自动追踪项目文件夹内的所有更改添加、修改、删除。
    • 尽管邮箱可以随便写，但最好还是和你代码托管平台例如 GitHub的邮箱保持一致，以免推送时出现问题。

  • 暂存、提交

    • 提交是 Git 的核心，是整个工作流最重要的步骤。一次提交就是一个记录节点。
    • 可以直接提交工作区 → 本地仓库，但不提倡。为了在提交前有一个缓冲因为提交了就会生成记录，可以将你所做的修改先暂存到暂存区。可以多次暂存，再一口气全部提交。
    • 暂存 ，将当前文件夹以及所有子文件夹中的更改全部记录到暂存区。 
      git add .

      暂存之后，更改会变成暂存的更改。

    • 提交 ，将暂存区所有的更改记录提交到本地仓库。 
      git commit -m "初始提交"
      -m 表示需要输入简明扼要的提交信息，概括一下这次提交主要更改了什么。注意：提交信息是必需的，也是提交的规范，不可省略；一般第1次提交的信息都写初始提交。
